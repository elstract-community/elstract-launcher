## 💡 Что это такое

**Elstract Launcher** — бесплатный open-source лаунчер для Windows, который позволяет собрать все твои игры в одном месте с красивым интерфейсом в стиле Steam.

**Зачем он нужен, если есть Steam?**

- Не все игры есть в Steam (GOG, Epic, itch.io, пиратки, собственные сборки)
- Steam требует аккаунт и интернет
- Elstract работает полностью **офлайн** после первоначальной настройки
- Ты сам владеешь своими данными
- Это open-source проект — ты можешь изменить буквально всё

## 📖 Подробное руководство пользователя

### Главная страница

При каждом запуске лаунчера открывается **главная страница**. Здесь находится:

- **Текущая версия лаунчера** (слева, в виде бейджа `v1.0.0`)
- **Кнопка "Open Library"** — переход в библиотеку игр
- **Кнопка "View on GitHub"** — открывает репозиторий в браузере
- **Карточка версии** (справа) — показывает текущую версию и статус обновлений:
  - 🟢 «Up to date» — у тебя актуальная версия
  - 🚀 «Update Available» — доступна новая версия, с кнопкой перехода к релизам
- **Секция Open Source** — описание проекта и ссылки на GitHub
- **Список функций** — краткое описание всех возможностей лаунчера
- **Лента релизов GitHub** — последние версии с датами выхода

### Библиотека игр

Перейди в **Library** через левую боковую панель.

**Интерфейс библиотеки:**

- **Сетка карточек** — каждая карточка показывает:
  - Обложку игры (или градиентный заменитель если обложки нет)
  - Название игры
  - Общее время игры (или живой счётчик сессии если игра запущена)
  - Значок `⚠ No EXE` если `.exe` ещё не привязан
  - Зелёный бейдж `▶ Running` если игра сейчас запущена
  - Звёздочку ⭐ если игра добавлена в избранное
- **Поиск/фильтр** — строка поиска в верхней части фильтрует игры по названию в реальном времени
- **Панель деталей** (справа) — появляется при клике на карточку:
  - Живой счётчик текущей сессии (зелёный блок, обновляется каждую секунду)
  - Общее время игры
  - Дата последнего запуска
  - Количество запусков
  - Дата добавления
  - Жанр, разработчик
  - Steam App ID
  - Путь к `.exe`
  - Описание игры
  - Кнопки **Play** / **Link EXE** и меню **⋮** с опциями

**Меню ⋮ (три точки):**
- `Link / Change EXE` — привязать или сменить исполняемый файл
- `Change Icon` — поставить свою обложку
- `Toggle Favourite` — добавить/убрать из избранного
- `Remove from Library` — удалить игру из библиотеки (файлы не удаляются)

### Добавление игры через Steam

1. Нажми **"+ Add Game"** в верхней части библиотеки
2. В открывшейся панели введи название игры в поле поиска
3. Нажми **"Search"** или клавишу **Enter**
4. В списке результатов найди нужную игру и **кликни по ней**
5. Лаунчер автоматически:
   - Загружает название, разработчика, издателя, описание, жанр, дату выхода
   - Скачивает обложку с Steam CDN и сохраняет локально
6. Появится диалог выбора `.exe` — укажи путь к исполняемому файлу игры
7. Игра появится в библиотеке готовой к запуску ✅

> **Где найти `.exe`?**  
> Обычно это `C:\Program Files (x86)\Steam\steamapps\common\ИМЯ_ИГРЫ\`.  
> Для не-Steam игр — папка куда ты установил игру.

---

### Добавление кастомной игры

Для игр не из Steam или любых других программ:

1. Нажми **"+ Add Game"**
2. Нажми кнопку **"🎮 Add Custom Game"**
3. Заполни форму:
   - **Game Name** (обязательно) — название игры
   - **Description** — описание (можно оставить пустым)
   - **Icon / Cover Image** — кликни на превью, чтобы выбрать файл изображения (PNG, JPG, BMP)
   - **Background / Banner** — баннер для панели деталей игры
   - **Developer** — разработчик
   - **Genre** — жанр
4. Нажми **"Add Game"**
5. Сразу откроется диалог выбора `.exe` — укажи путь к игре

> **Поддерживаемые форматы изображений:** PNG, JPG, JPEG, BMP, WEBP

### Запуск игры

**Способ 1:** Двойной клик по карточке игры в сетке

**Способ 2:** Клик по карточке → нажать кнопку **Play** в панели деталей

**Что происходит при запуске:**
1. Запускается `.exe` файл игры
2. Лаунчер начинает отсчёт сессии (синхронизирован с системными часами)
3. В Discord обновляется статус «Playing X»
4. Если в настройках включено «Minimize on launch» — лаунчер сворачивается
5. На карточке появляется зелёный бейдж «▶ Running»
6. В панели деталей показывается живой таймер текущей сессии

**При закрытии игры:**
1. Время сессии добавляется к общему времени игры
2. Discord статус сбрасывается на «In Elstract Launcher»
3. Библиотека автоматически сохраняется
4. Если в настройках включено «Close on game exit» — лаунчер закрывается

---

### Трекинг времени

Лаунчер отслеживает время автоматически:

- **Живой таймер сессии** — обновляется каждую секунду, синхронизирован с системными часами Windows. Показывается в формате `00m 00s` или `1h 23m 45s`
- **Общее время** — накапливается с каждой сессией, показывается на карточке в формате `12h 34m` или `45m`
- **Дата последнего запуска** — обновляется при каждом запуске
- **Счётчик запусков** — общее количество запусков игры

Все данные хранятся в `%AppData%\Elstract\library.json` и **никуда не отправляются**.

### Discord Rich Presence: Настройка

Discord Rich Presence позволяет показывать в профиле что ты сейчас запускаешь в Elstract.

**Шаг 1 — Создай Discord Application:**

1. Перейди на [discord.com/developers/applications](https://discord.com/developers/applications)
2. Нажми **"New Application"**
3. Введи название (например `Elstract Launcher`)
4. Скопируй **Application ID** (длинное число, например `1234567890123456789`)

**Шаг 2 — Добавь иконку (опционально):**

1. В разделе **"Rich Presence → Art Assets"** загрузи изображение
2. Назови его `elstract_logo` (именно так написано в коде)
3. Сохрани

**Шаг 3 — Вставь ID в код:**

1. Открой файл `Services/DiscordService.cs`
2. Найди строку:
   ```csharp
   private const string ClientId = "1234567890123456789";
   ```
3. Замени `1234567890123456789` на свой Application ID
4. Пересобери проект

**Шаг 4 — Включи в настройках:**

Запусти лаунчер → зайди в **Settings** → убедись что "Enable Discord Rich Presence" включён.

> **Примечание:** Discord должен быть запущен на компьютере. RPC работает через локальный сокет — данные в интернет не передаются.

**Что показывается в Discord:**
- **Idle:** `Browsing Library | Elstract Launcher`
- **В игре:** `Playing Cyberpunk 2077 | via Elstract Launcher` + иконка игры из Steam + таймер сессии

### Настройки

Перейди в **Settings** через левую панель.

#### Launch Behavior

| Настройка | По умолчанию | Описание |
|---|---|---|
| Minimize on game launch | ✅ Вкл | Сворачивает лаунчер когда ты запускаешь игру |
| Close launcher when game exits | ❌ Выкл | Закрывает лаунчер автоматически когда закрываешь игру |
| Track play time | ✅ Вкл | Включает/выключает запись времени игры |

#### Discord Rich Presence

| Настройка | По умолчанию | Описание |
|---|---|---|
| Enable Discord Rich Presence | ✅ Вкл | Показывать статус в Discord |
| Show game name in Discord status | ✅ Вкл | Показывать название игры (или просто "In Launcher") |

#### Data & Storage

Здесь отображаются пути к файлам данных:
- `%AppData%\Elstract\library.json` — библиотека игр
- `%AppData%\Elstract\Icons\` — кэш обложек из Steam

#### Кнопка Save Settings

Нажми **"Save Settings"** чтобы сохранить изменения.

## 🔨 Сборка из исходников

### Необходимые инструменты

- [.NET 8 SDK](https://dotnet.microsoft.com/download/dotnet/8.0)
- Windows 10 / 11
- IDE: [Visual Studio 2022](https://visualstudio.microsoft.com/) (рекомендуется), [JetBrains Rider](https://www.jetbrains.com/rider/), или VS Code с расширением C#

### Шаги

```bash
# 1. Клонируй репозиторий
git clone https://github.com/elstract-community/elstract-launcher.git
cd elstract-launcher

# 2. Восстанови NuGet пакеты
dotnet restore

# 3. Собери проект
dotnet build

# 4. Запусти
dotnet run --project ElstractLauncher
```

### Сборка релизного бинарника

```bash
# Self-contained (включает .NET runtime, ~80 МБ)
dotnet publish -c Release -r win-x64 --self-contained true -o ./publish

# Без runtime (нужен установленный .NET 8, ~10 МБ)
dotnet publish -c Release -r win-x64 --self-contained false -o ./publish
```

Результат в папке `./publish` — это `ElstractLauncher.exe` который можно распространять.

| `DiscordRichPresence` | Discord RPC интеграция |

## 🏛 Архитектура MVVM

Elstract строго следует паттерну **Model-View-ViewModel**:

```
┌──────────┐    binds to    ┌────────────┐    calls    ┌─────────┐    reads/writes    ┌──────┐
│  View    │ ─────────────► │ ViewModel  │ ──────────► │ Service │ ────────────────► │ Data │
│  (XAML)  │ ◄───────────── │   (C#)     │ ◄────────── │  (C#)   │ ◄──────────────── │(JSON)│
└──────────┘  INotifyPropCh └────────────┘  ObsColl    └─────────┘                   └──────┘
```

**Правила:**
- **View** — только XAML + минимальный code-behind (chrome, dialog helpers)
- **ViewModel** — вся логика UI, команды (`ICommand`), фильтрация, состояние
- **Service** — бизнес-логика без UI зависимостей (кроме `Dispatcher.Invoke`)
- **Model** — чистые данные с `INotifyPropertyChanged`

**Сервисы** — синглтоны, доступные через `App.Library`, `App.Discord` и т.д.  
Инициализируются в `App.OnStartup()` **до** создания главного окна.

## 🎨 Темизация

Все цвета определены в `Themes/ElstractTheme.xaml` как именованные ресурсы.

### Основная палитра

```xml
<!-- Розовый акцент -->
<Color x:Key="PinkPrimary">#FF1A7A</Color>   <!-- Основной розовый -->
<Color x:Key="PinkLight">#FF5FA3</Color>      <!-- Светлее (для текста) -->
<Color x:Key="PinkDark">#C4004F</Color>       <!-- Темнее (для эффектов) -->

<!-- Фоны -->
<Color x:Key="BgDeep">#0D0A10</Color>         <!-- Самый тёмный (тайтлбар) -->
<Color x:Key="BgBase">#130F18</Color>         <!-- Основной фон окна -->
<Color x:Key="BgCard">#1C1625</Color>         <!-- Фон карточек/панелей -->
<Color x:Key="BgSidebar">#160F1E</Color>      <!-- Фон сайдбара -->
<Color x:Key="BgInput">#221A2E</Color>        <!-- Фон инпутов -->

<!-- Текст -->
<Color x:Key="TextPrimary">#F0ECF8</Color>    <!-- Основной текст -->
<Color x:Key="TextSecondary">#A895C4</Color>  <!-- Вторичный текст -->
<Color x:Key="TextMuted">#5E4D7A</Color>      <!-- Приглушённый текст -->

<!-- Статусы -->
<Color x:Key="StatusGreen">#39D98A</Color>    <!-- Запущена/успех -->
<Color x:Key="StatusRed">#FF4757</Color>      <!-- Ошибка -->
<Color x:Key="StatusYellow">#FFC312</Color>   <!-- Предупреждение -->
```

### Создание своей темы

1. Скопируй `Themes/ElstractTheme.xaml` → `Themes/MyTheme.xaml`
2. Измени цвета
3. В `App.xaml` замени:
   ```xml
   <ResourceDictionary Source="Themes/ElstractTheme.xaml"/>
   ```
   на:
   ```xml
   <ResourceDictionary Source="Themes/MyTheme.xaml"/>
   ```
4. Пересобери проект

## 🔒 Данные и приватность

### Где хранятся данные

```
%AppData%\Elstract\
├── library.json      ← Библиотека игр (имена, пути, время игры, иконки)
├── settings.json     ← Настройки лаунчера
├── crash.log         ← Лог ошибок (пишется только при крашах)
└── Icons\            ← Кэш обложек скачанных с Steam CDN
```

### Что уходит в интернет

| Запрос | Когда | Зачем |
|---|---|---|
| Steam Store API | При поиске игры | Получить метаданные и обложку |
| Steam CDN | При добавлении Steam игры | Скачать обложку (1 раз, потом кэш) |
| GitHub API | При открытии главной страницы | Проверить наличие обновлений |
| Discord RPC | При запуске лаунчера | Локальный IPC сокет, **не интернет** |

**Ничего не отправляется без твоего действия.** Все Steam запросы — только когда ты сам нажимаешь «Search». GitHub запрос — только при открытии Home.

## 🤝 Как внести вклад

Любой вклад приветствуется — код, дизайн, переводы, документация, баг-репорты.

### Сообщить об ошибке

1. Проверь [существующие Issues](../../issues)
2. Открой [новый Issue](../../issues/new) и укажи:
   - Что произошло и как воспроизвести
   - Версия Windows и .NET
   - Содержимое `%AppData%\Elstract\crash.log`

### Предложить функцию

Открой [Issue](../../issues/new) с описанием идеи и зачем она нужна.

### Pull Request

1. **Fork** репозитория
2. Создай ветку: `git checkout -b feature/my-feature`
3. Сделай изменения (один PR = одна функция/фикс)
4. Убедись что проект собирается: `dotnet build`
5. Push и открой Pull Request в `main`

### Идеи для улучшений

- 🌍 **Переводы** — добавить русский/немецкий/японский UI
- 🎨 **Новые темы** — светлая тема, киберпанк, минимализм
- 🏷 **Категории игр** — папки/теги для организации библиотеки
- 📊 **Страница статистики** — графики времени игры по дням/месяцам
- 🔔 **Системный трей** — иконка в трее, уведомления
- 🛒 **GOG / Epic / itch.io** — поиск метаданных из других магазинов
- 📤 **Экспорт/импорт** — делиться библиотекой как JSON
- 🔍 **Автопоиск игр** — сканировать диск и находить установленные игры
- 🖼 **SteamGridDB** — больше обложек из стороннего источника
